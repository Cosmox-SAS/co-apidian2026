PROCEDIMIENTO DE ACTUALIZACION APIDIAN

Version php: 8.3
Documento: Procedimiento de actualización y/o sincronización del repositorio.

Notas generales
- Los comandos `apt-get` y `chmod` aplican en Linux (servidor) o dentro del contenedor Docker.
- Si estás en Windows (Laragon), esos comandos se ejecutan en WSL/Docker o donde corra Linux.
- Si aparecen conflictos de Git y necesitas descartar cambios locales: usar `git reset --hard origin/master`.

=====================================================================
1) ACTUALIZAR DESDE APIDIAN 2021/2022/2023/2024/2025 HACIA APIDIAN 2026
=====================================================================

1.1) IMPORTANTE (DOCKER)
Si la instalación corre en Docker, primero actualiza la imagen a la nueva versión.
Ejemplo (desde el directorio del proyecto donde está `docker-compose.yml`):

        IMPORTANTE: Antes de editar el `docker-compose.yml`, baja/detén los contenedores para que el cambio se aplique correctamente.
        (Ejemplo: `docker compose down` o `docker-compose down` en el directorio del .yml)

    1) nano docker-compose.yml
    2) Verifica que en la sección `php_api` tenga:
        php_api:
          build:
            context: .
            dockerfile: Dockerfile
          container_name: php_api
          working_dir: /var/www/html
          volumes:
            - ./:/var/www/html
          networks:
            - apinet
       (Si no aparece, cámbialo.)

1.2) CAMBIAR EL ORIGIN DEL REPOSITORIO (en el proyecto antiguo)
Dentro del directorio del proyecto (apidian2021/apidian2022/apidian2023/apidian2024/apidian2025):

    git config --global http.sslverify false
    git remote remove origin
    git remote add origin https://git.buho.la/facturalatam/co-apidian2026.git

1.3) VALIDAR/COMPLETAR LA CONFIGURACIÓN DE GIT
Editar `.git/config`:

    nano .git/config

Debe quedar así (si falta algo, agregarlo manualmente):

    [core]
        repositoryformatversion = 0
        filemode = false
        bare = false
        logallrefupdates = true
        symlinks = false
        ignorecase = true
    [remote "origin"]
        url = https://git.buho.la/facturalatam/co-apidian2026.git
        fetch = +refs/heads/*:refs/remotes/origin/*
    [branch "master"]
        remote = origin
        merge = refs/heads/master

1.4) SINCRONIZAR CÓDIGO

    git pull
    git pull origin master

Si aparecen conflictos:

    git reset --hard origin/master

1.5) DEPENDENCIAS / MIGRACIONES / POST-INSTALACIÓN

    composer clear-cache
    composer self-update 2.8.0
    rm composer.lock
    composer install

Luego:

    php artisan migrate --seed
    unzip storage.zip
    chmod -R 777 storage
    chmod -R 777 bootstrap/cache
    chmod -R 777 vendor/mpdf/mpdf
    php artisan config:cache && php artisan config:clear && php artisan cache:clear
    chmod 700 urn_on.sh
    ./urn_on.sh

Nota (Docker)
Se debera ajustar tambien el archivo docker-compose.yml y buscar el campo php_api, Debe tener esta estructura:

        IMPORTANTE: Antes de editar el `docker-compose.yml`, baja/detén los contenedores para que el cambio se aplique correctamente.
        (Ejemplo: `docker compose down` o `docker-compose down` en el directorio del .yml)

    php_api:
      build:
        context: .
        dockerfile: Dockerfile
      container_name: php_api
      working_dir: /var/www/html
      volumes:
        - ./:/var/www/html
      networks:
        - apinet

Esto vuelve a habilitar la opción de cargar fichas RUT.
